Atenção!

Este documento não contém a representação completa do curso sugerida pelo autor original do curso. Mas sim o exemplo de uso do que foi entendido no curso. Como o desafio propõe em seu enunciado:

A ideia deste Desafio de Projeto é elaborar um website, utilizando uma estrutura bem básica, utilizando tags aprendidas durante as aulas práticas:

  • Utilizar todas as tags explicadas nas aulas: <h1> até <h6>, <p>, <mark>, <small>, <i>, <u>, <strong>, <ol>, <ul>, <li>, <a>, <hr>, <sub>, <sup>, <blockquote>;
  • Utilizar novas tags sugeridas: <font>, <del>, <p>, <abbr> (a ideia é buscar estas tags na internet, entender como ela funciona e utilizar no texto).

Assim, até esta linha o foram usadas as seguintes tags HTML:

  • <H1> e <H2> para os títulos,
  • <SMALL> e <SUP> para o link "1" para o item 1 em "Referências".
  • <U> para sublinhar o texto "exemplo de uso".
  • <HR> para a linha divsória abaixo do título principal.
  • <BLOCKQUOTE> para apresentar a citação de um texto extraído de outro documento. Inclusive o atributo cite contém a URL da página "Entendendo o Desafio" dentro da DIO e pode não estar mais accessível.
  • <P> para marcar os parágrafos.
  • <I> para marcar o texto da citação com o estilo itálico.
  • <STRONG> para dar enfase e peso do tipo "negrito".
  • <UL> e <LI> para lista de 2 itens na citação.
  • <DEL> para marcar a exclusão da ocorrência redundante do texto da tag P. Mas também para a exclusão da tag FONT, pois esta tag foi descontinuada e seu uso não é mais recomendado. A referência para a tag FONT no site MDM avisa que a funcionalide desta tag pode cessar a qualquer momento.
  • <FONT> para colorir algums textos com a cor "Crimson" e alterar o tamanho do tipo, mas esta apresentação poderá não funcionar em seu navegador, pelo motivo citado no item anterior. Estilizar ou descrever a apresentação de texto é tarefa do código em CSS.
  • <A> para criar as ligações com outras páginas externas e a ligação do minúsculo "1" no primeiro título com o item "1" nas "Referências" no final deste documento.
  • A tag <ABBR> foi usada para marcar a abreviação da palavra "HTML" no parágrafo acima desta lista e em algumas ocorrências do nome do navegador e do editor no resto do documento. Porém, no parágrafo acima, a palávra "HTML" contém o atributo title com valor do nome expandido à qual refere-se esta abreviação. Se você estiver usando um mouse ou outro dispositivo apontador de tela, o nome expandido da abreviação aparecerá ao passar o cursor sobre a abreviação "HTML".

Com a tag <SUB>, posso marcar o número 2 no texto que representa uma molécula de água, ou seja, H2O.

Já o texto "CLIQUE AQUI PARA IR PARA OUTRA PÁGINA" é marcado com a tag <A> e carregará o conteúdo do arquivo outra-pagina.html nesta guia do navegador se for clicado.

Ferramentas utilizadas

Vamos preparar o ambient para iniciar o desenvolvimento de um site pequeno.

Navegador de Internet

Para isto a primeira ferramenta é o Navegador Google Chrome, que chamaremos de Chrome a partir de agora. Este navegador pode ser instalado ao seguir as instruções deste endereço: https://www.google.com/intl/pt-BR/chrome/.

Editor de textos

O editor de texto onde será escrito o código HTML utilizado aqui é o Visual Studio Code. Este é um editor gratuíto, open source, com várias funcionalides muito úteis e que pode ser executado em diversos sitemas operacionais.

No site do Visual Studio Code você encontrará as instruções necessárias para a instalação e uso deste editor.

Neste estudo, usaremos o nome reduzido do editor: VSCode.

Extensões do VSCode

O VSCode pode adquirir funcionalides extras com a instalação de extensões. Das várias extensões disponíveis para o editor no Marketplace do VSCode, a Live Server de Ritwick Dey será a primeira a ser instalada.

A extensão, Live Server adiciona na barra de status do editor um botão que cria um servidor HTML. O servidor usa o caminho do documento HTML em edição, como ponto de partida. Uma guia do navegador de internet padrão será aberta, exibindo o conteúdo do documento HTML. A cada alteração no código do documento o navegador atualizará a apresentação do documento.

Com o VSCode instalado, execute-o e então acesse o painel de extensões.

  1. Vá no menu View ou Visualizar e selecione a opção Extensions ou Extensões. O painel do gerenciador de extensões será aberto.
  2. No topo do painel gerenciador de extensões há uma caixa de texto (ou campo de formulário) onde pode ser feita a busca por texto dentro do Marketplace do VSCode. Escreva o nome da extensão ou palavras-chave relativas à extensão desejada. Neste momento, escreva o seguinte texto: live server. E então pressione Enter ou Return.
  3. Na lista de resultados da busca, abaixo do campo de busca, deverão aparecer alguns resultados na forma de botões que abrem uma página descrevendo o ítem. Um destes itens deve descrever a extensão chamada Live Server, cujo o autor é Ritwick Dey.
  4. Na página de descrição da extensão, clique no botão Install ou Instalar
